224 // Require that user allocate at least 550MB for block & undo files (blk???.dat and rev???.dat)
225 // At 1MB per block, 288 blocks = 288MB.
226 // Add 15% for Undo data = 331MB
227 // Add 20% for Orphan block rate = 397MB
228 // We want the low water mark after pruning to be at least 397 MB and since we prune in
229 // full block file chunks, we need the high water mark which triggers the prune to be
230 // one 128MB block file + added 15% undo data = 147MB greater for a total of 545MB
231 // Setting the target to > than 550MB will make it likely we can respect the target.
